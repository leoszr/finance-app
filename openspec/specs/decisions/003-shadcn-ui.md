# ADR 003 - shadcn/ui com Tailwind

## Status
Aceito

## Contexto
Precisamos de componentes acessiveis, customizaveis e consistentes com mobile-first.

## Decisao
Adotar shadcn/ui sobre Tailwind como base de componentes.

## Racional
- Componentes acessiveis e pragmaticos para formularios e dialogs.
- Customizacao direta no codigo do repositorio.
- Alinhado ao stack escolhido para Next.js.

## Consequencias
- Biblioteca de UI centralizada em `components/ui`.
- Evolucao visual depende de governanca interna de componentes.
